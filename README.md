## 2.1 linked list에 관한 함수 



###   2.1.1 create_list 

### 이 함수는 사용자에게 10자리의 ID를 사용자에게 받아 각 노드당 1개의 값을 집어 넣고 10개의 노드로 구성된 linked list를 만들어 주는 함수이다.
### 이 함수가 중요한 이유는 이 함수로 생성한 linked list에 이후에 설명할 여러 함수들을 이용하기 때문이다.
### 예를 들어 10자리 ID '2023171020'을 입력해줄 경우 이를 10개의 노드에 각각의 값을 한자리씩 나누어 저장해 2,0,2,3,1,7,1,0,2,0으로 구성된 list 구조를 만들어 낸다.  

###   2.1.2 create_node 

### 이 함수는 기본적인 함수로 사용자가 원하는 1개의 값을 담은 1개의 노드를 생성하는 함수이다.
### 이후 이 함수는 새로 생성한 노드의 포인터를 반환해주는 역할을 수행한다. 
### 이 함수가 중요한 이유는 add node와 같은 다른 함수들 안에서 사용되기 때문이다.

### 2.1.3 print_list 

### 이 함수는 구성된 linked list안의 값들을 하나 하나 list의 앞부터 뒤까지 출력해주는 함수이다. 
### 이 함수가 중요한 이유는 우리가 이후에 나올 여러 add_node, remove_node 같은 함수들이 잘 실행 되었는지 확인해주는 역할을 수행하기 때문이다.
### 또 이 함수를 통해 linked list라는 자료구조의 현재 저장하고 있는 값을 알 수 있기에 매우 유용한 함수라고 볼 수 있다.

### 2.1.4 remove_node

### 이 함수는 특정 linked list에서 삭제할 위치(node_location)를 입력해주면 그 위치에 해당하는 노드 한개를 삭제 해주는 함수이다. 
### 이 함수는 프로그램 사용자가 자료구조에서 특정한 data를 없에고 싶을 경우에 이용하면 된다.  
### 이해를 돕기 위해 직접 예시를 들자면 2, 0, 2, 3, 4 가 저장되어있는 linked list 에서 삭제할 node_location을 2로 준다면 
### 이후 이 linked list 자료구조에는 0이 삭제되고 2, 2, 3, 4 만 남게된다.

### 2.1.5 add_node

### 이 함수는 특정 linked list에서 주어진 위치(loc)에 원하는 값(newVal)을 넣어주고 싶을때에 사용하는 함수이다. 
### 사용자가 자료 구조에 새로운 데이타를 추가하고 싶은 상황에 이 함수를 사용하면 된다.   
### 이해를 돕기 위해 직접 예시를 들자면 2, 0, 2, 3, 4 가 저장되어있는 linked list 에서 추가할 위치(loc)을 2로 주고 newVal 값을 6으로 준다면   
### 이후 이 linked list 자료구조에는 6이 추가되어  2, 6, 0, 2, 3, 4 가 저장된다.

### 2.1.6 count_node_values

### 이 함수는 특정 linked list에서 찾고 싶은 값(value)을 입력하면 그 값이 해당 li의 여러 operation들을 실행해 볼 수 있는 프로그램이다. 
### 사용자에게 0부터 20까지으 정수를 입력받아 (참고로 0은 프로그램 종료이다) 사용자가 원하는 종류의 자료구조에 해당 operation을 수행할 수 있도록 도와준다.
### 이 프로그램을 통해 사용자는 원하는 형태의 자료구조로 data의 조작 및 관리를 할 수 있게 된다. 

### 2.1.7 convert_to_circularLinkedList

### 이 함수는 자료구조의 형태를 바꾸어주는 함수로 볼 수 있다. 
### 현재 linked list가 circular하지 않은 list일 경우에 list의 head와 tail이 각각 자신의 끝과 자신의 처음을 가르키게 해주어 circular list의 형태로 변환해줄 수 있다. 
### 이 함수는 사용자가 non-circular list로는 원하는 기능을 편리하게 수행하지 못할 경우에 자료 구조를 circular linked list의 형태로 바꾸어주어 
### data를 조금더 유연하게 다룰 수 있도록 도와주는 역할을 수행한다. 

### 2.1.8 remove_list 

### 이 함수는 linked list에 값이 있다면 그 안에 있는 모든 값들을 없에주고 linked list라는 자료구조 자체를 아예 없애주는 역할을 수행한다. 
### 예를 들어 2, 6, 0, 2, 3, 4 로 구성된 linked list에 remove_list를 해주면 이 안의 값들을 하나 하나 다 없애주고 결국에는 linked list 자체까지 없애 버린다.

## 2.2 stack에 관한 함수 



### 2.2.1 print_stack

### 이 함수는 현재 stack안에 있는 값들을 하나하나 출력해주는 함수이다.
### stack이라는 자료구조의 현재 상태를 보여주는 함수로 debugging 과정에서 유용하게 이용될 수 있다.


### 2.2.2 create_stack

### 이 함수는 stack이라는 자료구조의 기본틀을 만들어주는 함수로 비어있는 stack 하나를 만들어 주는 역할을 수행한다. 
### 이 함수로 stack을 만들어준 후에 그안에 push나 pop등을 해줄 수 있다. 

### 2.2.3 check_empty_stack, check_full_stack

### 이 함수는 stack이라는 자료구조가 현재 어떤 상태인지 알려주는 함수이다. 
### check_empty_stack은 stack이 비어있으면 알려주는 함수이고
### check_full_stack은 stack이 다차있어 더이상 추가할 수 없음을 알려주는 함수이다.

### 2.2.4 push

### 이 함수는 stack이라는 자료구조에 data를 추가할때 이용하는 함수이다.
### 추가한 데이터는 항상  LIFO 방식에 따라 stack의 top에 추가된다. 그리고 추가 이후에는 추가한 값을 반환해 주는 함수이다.
### 예를 들어 2,3,4 (2가 top이다)에 1을 push해주면 stack에 1,2,3,4(1이 새로운 top이다)의 형태로 저장된다. 

### 2.2.5 pop

### 이 함수는 stack이라는 자료구조에 data를 삭제할때 이용하는 함수이다.
### 삭제할 데이터는 항상  LIFO 방식에 따라 stack의 top에 있는 것이 삭제된다. 그리고 삭제한 값을 return해 준다. 
### 예를 들어 2,3,4 (2가 top이다)에 2를 pop해주면 stack에 3,4(3이 새로운 top이다)의 형태로 저장된다. 그리고 pop된 값인 2를 return 한다. 

### 2.2.6 peek

### 이 함수는 stack이라는 자료구조에 data를 삭제하지 않으면서 top에 있는 값만을 볼고 싶을때에 이용하는 함수이다.
### 예를 들어 2,3,4 (2가 top이다)에 peek해주면 top에 있는 2가 이 함수의 최종 return값이 되고 이후 main 함수에서 print 문을 통해 이 값을 확인해줄 수 있다.

### 2.2.7 remove_stack 

### 이 함수는 stack에 값이 있다면 그 안에 있는 모든 값들을 pop해주고 stack이라는 자료구조 자체를 아예 없애주는 역할을 수행한다. 
### 예를 들어 2, 6, 0, 2, 3, 4 로 구성된 stack에 remove_stack를해주면 이 안의 값들을 하나 하나 다 없애주고 결국에는 stack 자체까지 없애 버린다.

### 2.2.8 infix_to_postfix

### 이 함수는 stack이라는 자료구조의 성질을 이용하여 infix로 표현된 수식을 postfix의 형태로 바꾸어주는 역할을 수행한다. 
### 예를 들어 13+8^9-6(6+89)-237라는 infix수식을 입력하면 이를 13 8 9 ^ + 6 6 89 + - 237 - 라는 postfix 수식으로 변환하여 출력해주는 함수이다.

## 2.3  queue에 관한 함수 



### 2.3.1 print_queue
### 이 함수는 queue의 front에서 rear까지 현재 queue에 있는 값들을 순서대로 출력해주는 함수이다. 
### queue이라는 자료구조의 현재 상태를 보여주는 함수로 debugging 과정에서 유용하게 이용될 수 있다.

### 2.3.2 create_queue
### 이 함수는 queue 라는 자료구조의 기본틀을 만들어주는 함수로 실행을 하고나면 
### queue size가 0이고 front와 rear이 NULL을 가르키는 비어있는 queue 구조체 한 개가 생성된다. 
### 그리고 최종적으로 이렇게 형성된 구조체의 포인터를 반환해주는 함수이다.

### 2.3.3 is_empty_queue
### 이 함수는 현재 queue의 head가 가르키는 값이 없는지를 통해 queue가 비어있는지 아닌지를 구분해주는 함수이다.
### 만약 비어있을 경우 1을 return 하고, 그 외의 경우에 0을 return해준다. 
### 이 함수는 나중에 dequeue 함수를 사용하는 과정에서 underflow 상황을 확인하는데 활용된다. 

### 2.3.4 enqueue
### 이 함수는 현재 queue라는 자료구조의 특징인 FIFO을 유지시키기위해 queue의 맨끝인 rear 부분에 input으로 받은 새로운 값(item)을 추가해주는 함수이다.  
### 예를 들어 현재 2,3,4,5 (front가 2고 rear이 5)의 데이터들이 저장된 queue 자료구조가 있으면 item 6을 enqueue를 해줄 경우에 
### queue에 저장된 데이터는 2,3,4,5,6 (front가 2고 rear이 6)인 queue가 된다. 

### 2.3.5 dequeue

### 이 함수는 현재 queue라는 자료구조의 특징인 FIFO을 유지시키기위해 queue의 맨앞인 front 부분에 있는 값을 삭제해주고 그 삭제한 값을 반환해주는 함수이다.  
### 예를 들어 현재 2,3,4,5 (front가 2고 rear이 5)의 데이터들이 저장된 queue 자료구조가 있으면 dequeue를 해줄 경우에 
### queue에 저장된 데이터는 3,4,5 (front가 3고 rear이 5)인 queue가 된다. 그리고 이함수는 dequeue한 item인 2를 최종적으로 반환해 준다. 

### 2.3.6 remove_queue
### 이 함수는 queue 자료구조에 더이상 데이터가 없을때까지 dequeue를 반복해주고 
### 최종적으로 queue의 기본 틀을 담고있던 구조체까지 없애 queue라는 자료구조를 완벽하게 삭제해주는 역할을 수행하는 함수이다.
